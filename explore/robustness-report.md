## 问题诊断

### 问题1: 参数敏感性高 (CV 41.9%)

**原因**: 出场阈值 EXIT_THRESHOLD 对收益影响太大

**证据**:
- EXIT_THRESHOLD=-0.02 → +1128.4%
- EXIT_THRESHOLD=-0.03 → +732.7%
- 仅0.01的差异导致 395.7pp 的收益变化

**解决方向**:
1. 找更稳定的参数区域
2. 使用参数区间而非单点
3. 集成多个参数组合

### 问题2: 依赖少数大赚交易

**原因**: 21笔交易中，少数大赚交易贡献大部分收益

**证据**:
- 交易10: +83.7% (2024主升浪)
- 交易12: +48.7% (2025冲顶)
- 这2笔占总收益的 78%

**解决方向**:
1. 增加交易次数，分散收益来源
2. 降低对单笔交易的依赖
3. 更敏感的入场信号

### 问题3: 2024年过拟合嫌疑

**原因**: 交叉验证显示2024年收益远高于其他年份

**证据**:
- 2024年: 765.54%
- 2025年: 73.80%
- 差距 10.36x

**解决方向**:
1. 使用更多2022-2023年数据训练
2. 降低对2024年牛市的依赖
3. 测试在2022年熊市的表现

### 问题4: 部分窗口无交易

**原因**: 入场条件太严格，导致某些窗口无信号

**证据**:
- 2022 H2: 0笔 (熊市)
- 2024 Q4: 0笔 (冲顶期)

**解决方向**:
1. 放宽入场条件
2. 增加入场信号类型
3. 允许在特定市场环境下主动不交易

## 优化建议

### 优先级1: 参数稳定性优化

**目标**: CV < 30%

**方法**:
1. 找参数平坦区域 (收益 > 最优80%的区域)
2. 使用参数区间而非单点
3. 测试参数组合的稳定性

**具体步骤**:
```
1. 扫描 EXIT_THRESHOLD: [-0.025, -0.020, -0.015, -0.010]
2. 找到收益 > 900% 的区域
3. 选择该区域中最稳定的参数
```

### 优先级2: 增加交易次数

**目标**: 从21笔增加到30+笔

**方法**:
1. 放宽入场条件 (CB_THRESHOLD: 0.65 → 0.70)
2. 增加入场信号类型
3. 缩短持仓周期

**具体步骤**:
```
1. 测试 CB_THRESHOLD: [0.65, 0.70, 0.75]
2. 测试 ENTRY_MOM: [2, 3, 4]
3. 选择交易次数最多且收益稳定的组合
```

### 优先级3: 集成学习

**目标**: 降低对单参数的依赖

**方法**:
1. 训练3-5个不同参数的变体
2. 投票机制决定交易
3. 分散参数风险

**具体步骤**:
```
变体1: EXIT_THRESHOLD=-0.02 (当前最优)
变体2: EXIT_THRESHOLD=-0.025 (次优)
变体3: EXIT_THRESHOLD=-0.015 (激进)

入场: ≥2个变体发出信号
出场: ≥2个变体发出信号
```

### 优先级4: 动态止损

**目标**: 降低最大回撤

**方法**:
1. ATR-based止损
2. 根据波动率调整
3. 保护利润

**具体步骤**:
```
止损 = max(固定止损, ATR止损)
ATR止损 = 入场价 - 2 × ATR(20)
固定止损 = -25%
```
